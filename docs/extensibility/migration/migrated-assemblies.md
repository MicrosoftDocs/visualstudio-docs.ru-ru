# <a name="sdk-reference-packages"></a>Эталонные пакеты SDK

[!INCLUDE [preview-note](../includes/preview-note.md)]

Проще всего создать расширения Visual Studio с помощью ссылки на [ `Microsoft.VisualStudio.Sdk` пакет NuGet](https://www.nuget.org/packages/microsoft.visualstudio.sdk).
Этот пакет предназначен для Visual Studio 2017 (15,0), Visual Studio 2019 (16,0, 16,9) и теперь Visual Studio 2022.

В зависимости от расширения Добавление дополнительных пакетов пакета SDK VS, которые не включены в указанный выше мета-пакет, может потребоваться.
При указании ссылок на другие пакеты SDK эти пакеты могут различаться в основных версиях VS.

Обратите внимание, что многие сборки взаимодействия были внедрены до Visual Studio 2022. Начиная с Visual Studio 2022 внедрение больше не требуется или не поддерживается.
Используйте *ссылки на* сборки взаимодействия вместо их связывания.

В следующей таблице представлено сопоставление сборок или пакетов, которые ранее расширение Visual Studio 2022 могут ссылаться на новый идентификатор пакета для ссылки при разработке для Visual Studio 2022. В некоторых случаях сборки теперь доступны в пакетах NuGet, которые ранее были доступны только в локальной установке Visual Studio.
