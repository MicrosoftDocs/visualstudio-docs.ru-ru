# <a name="test-area-4-check-in"></a>Область тестирования 4. Запись после изменения
Эта область тестирования подключаемого модуля системы управления версиями охватывает отправку обновленных элементов в хранилище версий с помощью команды **возврата** .

## <a name="command-menu-access"></a>Доступ к меню команд
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]В тестовых случаях используются следующие пути к меню интегрированной среды разработки.

##### <a name="check-in"></a>Вернуть:
 **Файл**, **система управления версиями**, **Возврат**.

 **Файл**, **записать после записи**.

 Контекстное меню, **Возврат**.

## <a name="common-expected-behavior"></a>Типичное ожидаемое поведение

- Проекты и файлы, добавленные в решение или проект в системе управления версиями, отображаются в диалоговых окнах **Возврат** и **Ожидающие возвраты** .

- После возврата добавленные элементы появятся в системе управления версиями.

- После возврата обновленные элементы будут иметь правильную версию в хранилище.

## <a name="test-cases"></a>Тестовые случаи
 Ниже приведены конкретные тестовые случаи для области тестирования checked.

### <a name="case-4a-modified-items"></a>Случай 4а. Измененные элементы
 Описывает использование действия возврат для обновления файла в системе управления версиями, который был изменен.

|Действие|Шаги теста|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|Изменение текстового файла, который был извлечен, только возврат из файла (диалоговое окно "**Возврат** ")|1. Создайте новый проект с текстовым файлом.<br />2. Добавьте решение в систему управления версиями.<br />3. Извлеките и измените текстовый файл.<br />4. возврат с помощью диалогового окна "возврат" (**файл**, **система управления версиями**, **Возврат после записи**).|Типичное ожидаемое поведение.|
|Изменение извлеченного текстового файла, возврат только файла (окно "**извлеченные** файлы")|1. Создайте новый проект с текстовым файлом.<br />2. Добавьте решение в систему управления версиями.<br />3. Извлеките и измените текстовый файл.<br />4. запись в репозиторий с помощью окна **извлеченных параметров** .|Типичное ожидаемое поведение.|

### <a name="case-4b-adding-files"></a>Случай 4б. Добавление файлов
 При добавлении файла в проект или элемент в решение проект или решение также должны измениться. Таким образом, родительский файл также извлекается и должен быть возвращен для завершения добавления.

|Действие|Шаги теста|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|Добавление текстового файла и запись всех элементов (диалоговое окно «**Возврат** »)|1. Создайте новый проект.<br />2. Добавьте решение в систему управления версиями.<br />3. Добавьте в проект текстовый файл.<br />4. при появлении запроса примите условия поиска проекта.<br />5. Выберите решение в **Обозреватель решений**.<br />6. возврат из диалогового окна " **Возврат** ".|Типичное ожидаемое поведение.|
|Добавление текстового файла и запись всех элементов (окно "**возвраты** ")|1. Создайте новый проект.<br />2. Добавьте решение в систему управления версиями.<br />3. Добавьте в проект текстовый файл.<br />4. при появлении запроса примите условия поиска проекта.<br />5. Возврат решения из окна **ожидающих возвратов** .|Типичное ожидаемое поведение|

### <a name="case-4c-adding-projects"></a>Случай 4в. Добавление проектов
 При добавлении проекта в решение его также необходимо изменить. Поэтому файл решения также извлекается и должен быть возвращен для завершения добавления.

|Действие|Шаги теста|Ожидаемые результаты для проверки|
|------------|----------------|--------------------------------|
|Добавление проекта в пустое решение в системе управления версиями (диалоговое окно «**Возврат** »)|1. Создайте пустое решение.<br />2. Добавьте решение в систему управления версиями.<br />3. Добавьте новый проект.<br />4. при появлении запроса примите условия поиска решения.<br />5. возврат из диалогового окна " **Возврат** ".|Типичное ожидаемое поведение.|
|Добавление проекта в пустое решение в системе управления версиями (окно "**незавершенные возвраты** ")|1. Создайте пустое решение.<br />2. Добавьте решение в систему управления версиями.<br />3. Добавьте новый проект.<br />4. при появлении запроса примите условия поиска решения.<br />5. Возврат решения из окна **ожидающих возвратов** .|Типичное ожидаемое поведение.|
